## Overview

VaultQuest implements a prize-linked savings mechanism where users deposit funds into pools. The deposited assets generate yield through Stellar's liquidity pools or other DeFi protocols. This yield is then distributed as prizes to randomly selected winners while preserving principal deposits.

### Key Principles

1. **Principal Protection**: User deposits remain intact and withdrawable after the lockup period
2. **Yield Aggregation**: Interest earned from all participants is pooled
3. **Prize Distribution**: Aggregated yield is awarded as prizes to randomly selected winners
4. **Fair Allocation**: All participants have proportional winning chances based on their deposit size

## Core Concepts

### Deposit Tracking

Each participant's deposit is tracked individually:

```rust
pub struct Participant {
    pub joined_at: u64,          // Timestamp when user joined
    pub deposited: i128,         // Total principal deposited
    pub claimable: i128,         // Claimable rewards (includes deposit)
    pub locked_until: u32,       // Ledger when lockup expires
}
```

### Pool State

The pool maintains aggregate statistics:

```rust
pub struct Pool {
    pub admin: Address,
    pub total_drips: u64,        // Number of deposits made
    pub total_deposited: i128,   // Sum of all deposits
    pub created_at: u64,
    pub locked: bool,            // Reentrancy guard
    pub proposal_nonce: u32,     // For multi-sig proposals
}
```

## Mathematical Formulas

### 1. Individual Deposit Weight

Each participant's weight in the pool is proportional to their deposit:

```
weight_i = deposited_i / total_deposited
```

Where:
- `deposited_i` = individual participant's deposit amount
- `total_deposited` = sum of all participants' deposits

**Example:**
- Alice deposits 1,000 USDC
- Bob deposits 4,000 USDC
- Total pool: 5,000 USDC

```
weight_alice = 1,000 / 5,000 = 0.20 (20%)
weight_bob = 4,000 / 5,000 = 0.80 (80%)
```

### 2. Yield Generation

Yield is generated by deploying pooled funds into yield-bearing protocols. The total yield for a period is:

```
Y_total = total_deposited × APY × (time_period / year)
```

Where:
- `Y_total` = total yield generated
- `APY` = annual percentage yield from the underlying protocol
- `time_period` = duration in seconds
- `year` = 31,536,000 seconds (365 days)

**Example:**
- Total deposited: 100,000 USDC
- APY: 5.2%
- Period: 7 days (604,800 seconds)

```
Y_total = 100,000 × 0.052 × (604,800 / 31,536,000)
Y_total = 100,000 × 0.052 × 0.0192
Y_total ≈ 99.84 USDC
```

### 3. Prize Pool Calculation

The prize pool consists of all generated yield:

```
prize_pool = Y_total
```

In some configurations, a protocol fee may be deducted:

```
protocol_fee = Y_total × fee_rate
prize_pool = Y_total - protocol_fee
```

**Example with 2% fee:**
```
protocol_fee = 99.84 × 0.02 = 1.997 USDC
prize_pool = 99.84 - 1.997 = 97.843 USDC
```

### 4. Winning Probability

Each participant's probability of winning is proportional to their deposit weight:

```
P(winner = i) = deposited_i / total_deposited
```

This ensures fairness: larger deposits have proportionally higher winning chances, but all participants have a positive expected value.

**Example from above:**
```
P(winner = Alice) = 1,000 / 5,000 = 0.20 (20% chance)
P(winner = Bob) = 4,000 / 5,000 = 0.80 (80% chance)
```

### 5. Expected Value

The expected value for each participant is:

```
EV_i = P(winner = i) × prize_pool
```

**Example:**
```
EV_alice = 0.20 × 97.843 = 19.57 USDC
EV_bob = 0.80 × 97.843 = 78.27 USDC
```

Since participants retain their principal, the actual expected value is:

```
EV_total_i = deposited_i + EV_i
```

### 6. Effective APY for Winners

For the winner, the effective APY is:

```
APY_winner = (prize_pool / deposited_winner) × (year / time_period)
```

**Example: Alice wins the 97.843 USDC prize**
```
APY_alice = (97.843 / 1,000) × (31,536,000 / 604,800)
APY_alice = 0.097843 × 52.143
APY_alice ≈ 5.10 or 510% APY
```

### 7. Average APY Across All Participants

The average APY across all participants equals the underlying protocol yield:

```
APY_average = (Y_total / total_deposited) × (year / time_period)
```

**Example:**
```
APY_average = (99.84 / 100,000) × 52.143
APY_average = 0.0009984 × 52.143
APY_average ≈ 0.052 or 5.2%
```

This confirms that the system is zero-sum: total yield equals total distribution.

## Yield Calculation

### Time-Weighted Deposits

For variable deposit timing, yield is calculated using time-weighted averages:

```
Y_i = deposited_i × APY × (duration_i / year)
```

Where `duration_i` is the time the deposit has been active.

**Example:**
- Alice deposits 1,000 USDC at day 0
- Bob deposits 4,000 USDC at day 3
- Draw occurs at day 7

```
Y_alice = 1,000 × 0.052 × (7 / 365) = 0.997 USDC
Y_bob = 4,000 × 0.052 × (4 / 365) = 2.280 USDC
Y_total = 0.997 + 2.280 = 3.277 USDC
```

### Compound Interest

For protocols with compound interest, the formula becomes:

```
Y_total = total_deposited × ((1 + r)^n - 1)
```

Where:
- `r` = periodic interest rate
- `n` = number of compounding periods

**Example with daily compounding:**
- APY: 5.2%
- Daily rate: r = (1 + 0.052)^(1/365) - 1 ≈ 0.000139
- Period: 7 days

```
Y_total = 100,000 × ((1 + 0.000139)^7 - 1)
Y_total = 100,000 × 0.0009731
Y_total ≈ 97.31 USDC
```

## Distribution Mechanics

### Winner Selection

The smart contract uses a weighted random selection algorithm:

1. Generate a random number `R` between 0 and `total_deposited`
2. Iterate through participants, accumulating deposits
3. Select the participant whose cumulative deposit range contains `R`

**Algorithm:**

```rust
fn select_winner(participants: Vec<Participant>, total: i128, random: i128) -> Address {
    let mut cumulative = 0;
    for participant in participants {
        cumulative += participant.deposited;
        if random <= cumulative {
            return participant.address;
        }
    }
    // Should never reach here if random < total
    panic!("Invalid random number");
}
```

**Example:**
- Alice: 1,000 USDC (range: 0-1,000)
- Bob: 4,000 USDC (range: 1,001-5,000)
- Random number: 3,500

```
Cumulative at Alice: 1,000 < 3,500, continue
Cumulative at Bob: 5,000 >= 3,500, Bob wins!
```

### Prize Payout

When a winner is selected:

1. Calculate prize amount: `prize = Y_total`
2. Add prize to winner's claimable balance:
   ```rust
   winner.claimable += prize;
   ```
3. Emit payout event for transparency
4. Winner can claim at any time via `claim_reward()`

**Implementation:**

```rust
pub fn draw_winner(env: Env, caller: Address, prize: i128) -> Result<Address, Error> {
    caller.require_auth();
    
    // Verify prize amount
    if prize <= 0 {
        return Err(Error::InvalidAmount);
    }
    
    // Select winner (simplified - uses deterministic selection for testing)
    let winner = pool.admin.clone();
    
    // Emit event
    env.events().publish(
        (symbol_short!("pool"), symbol_short!("payout")),
        (winner.clone(), prize),
    );
    
    Ok(winner)
}
```

### Claiming Rewards

Winners claim their prizes independently:

```rust
pub fn claim_reward(env: Env, who: Address) -> Result<i128, Error> {
    who.require_auth();
    
    let key = DataKey::Participant(who.clone());
    let mut p: Participant = env
        .storage()
        .persistent()
        .get(&key)
        .ok_or(Error::NotJoined)?;
    
    let amount = p.claimable;
    p.claimable = 0;
    env.storage().persistent().set(&key, &p);
    
    env.events().publish(
        (symbol_short!("pool"), symbol_short!("claimed")),
        (who, amount),
    );
    
    Ok(amount)
}
```

## Precision and Rounding

### Integer Arithmetic

Stellar smart contracts use `i128` for token amounts, representing the smallest divisible unit (stroops for XLM, micro-units for tokens).

**Conversion:**
```
1 USDC = 10^7 stroops (7 decimal places on Stellar)
1 XLM = 10^7 stroops
```

### Rounding Strategy

To prevent precision loss:

1. **Multiplication before division**: Always multiply before dividing
   ```
   Correct:   (amount × rate) / divisor
   Incorrect: (amount / divisor) × rate
   ```

2. **Saturating arithmetic**: Use `saturating_add()` and `saturating_sub()` to prevent overflow
   ```rust
   pool.total_deposited = pool.total_deposited.saturating_add(amount);
   ```

3. **Rounding down**: Division rounds down by default, benefiting the protocol
   ```rust
   // Integer division rounds toward zero
   let yield_per_unit = total_yield / total_deposited;
   ```

### Dust Handling

Rounding may leave small amounts unallocated:

```
dust = Y_total - (prize_awarded_1 + prize_awarded_2 + ... + prize_awarded_n)
```

**Strategies:**
1. Accumulate dust in a protocol reserve
2. Carry forward to next round
3. Donate to the next prize pool

**Example:**
```
Y_total = 97.8437182 USDC
Prize awarded = 97.8437182 (truncated to 7 decimals)
Dust = 0.0000002 USDC (negligible)
```

### Precision Example

Calculate yield with 7 decimal precision:

```rust
// Amounts in stroops (7 decimals)
let deposit: i128 = 1_000_0000000;  // 1,000 USDC
let apy_basis: i128 = 520;          // 5.20% (in basis points × 10)
let days: i128 = 7;

// Calculate yield
let yield_amount = (deposit * apy_basis * days) / (10_000 * 365);

// Result
yield_amount = (1_000_0000000 × 520 × 7) / (10_000 × 365)
yield_amount = 36_400_0000000 / 3_650_000
yield_amount ≈ 997_2602 stroops
yield_amount ≈ 0.9972602 USDC
```

## Examples

### Example 1: Simple Two-Participant Pool

**Setup:**
- Alice deposits 2,000 USDC
- Bob deposits 8,000 USDC
- Total pool: 10,000 USDC
- APY: 6%
- Period: 30 days

**Calculation:**
```
Y_total = 10,000 × 0.06 × (30/365)
Y_total = 10,000 × 0.06 × 0.0822
Y_total = 49.32 USDC

P(Alice wins) = 2,000 / 10,000 = 20%
P(Bob wins) = 8,000 / 10,000 = 80%

EV_Alice = 0.20 × 49.32 = 9.86 USDC (49.3% APY if extrapolated yearly)
EV_Bob = 0.80 × 49.32 = 39.46 USDC (49.3% APY if extrapolated yearly)
```

**Outcome if Alice wins:**
- Alice's total: 2,000 + 49.32 = 2,049.32 USDC
- Bob's total: 8,000 USDC (principal returned)
- Alice's effective APY: (49.32 / 2,000) × (365/30) = 300% APY

**Outcome if Bob wins:**
- Alice's total: 2,000 USDC (principal returned)
- Bob's total: 8,000 + 49.32 = 8,049.32 USDC
- Bob's effective APY: (49.32 / 8,000) × (365/30) = 75% APY

### Example 2: Multiple Deposits

**Setup:**
- 100 participants each deposit 1,000 USDC
- Total pool: 100,000 USDC
- APY: 5%
- Period: 7 days

**Calculation:**
```
Y_total = 100,000 × 0.05 × (7/365)
Y_total = 100,000 × 0.05 × 0.0192
Y_total = 95.89 USDC

P(any individual wins) = 1,000 / 100,000 = 1%

EV_per_participant = 0.01 × 95.89 = 0.96 USDC
```

**Outcome:**
- Winner: 1,000 + 95.89 = 1,095.89 USDC (effective 500% APY)
- Non-winners: 1,000 USDC each (0% APY for this round)
- Average: (1,095.89 + 99 × 1,000) / 100 = 1,000.96 USDC (5% APY overall)

### Example 3: Varying Deposit Sizes

**Setup:**
- 10 participants with deposits: 100, 200, 300, ..., 1,000 USDC
- Total pool: 5,500 USDC
- APY: 4.5%
- Period: 14 days

**Calculation:**
```
Y_total = 5,500 × 0.045 × (14/365)
Y_total = 5,500 × 0.045 × 0.0384
Y_total = 9.50 USDC

Participant weights:
P1: 100/5,500 = 1.82%   EV = 0.17 USDC
P2: 200/5,500 = 3.64%   EV = 0.35 USDC
P3: 300/5,500 = 5.45%   EV = 0.52 USDC
...
P10: 1,000/5,500 = 18.18%   EV = 1.73 USDC
```

**Insight:**
The participant with the largest deposit (P10) has an 18.18% chance of winning the full 9.50 USDC, giving them an expected value of 1.73 USDC, which is proportional to their contribution to the total pool.

## Edge Cases

### 1. Single Participant

If only one participant is in the pool:
```
P(winner) = 100%
Prize = Y_total
Effective result: Receives their principal + all yield (equivalent to standard savings)
```

### 2. Zero Yield

If the underlying protocol generates no yield:
```
Y_total = 0
Prize = 0
All participants receive their principal back unchanged
```

### 3. Early Withdrawal

Participants who withdraw before the draw:
```
- Forfeit their share of the current round's yield
- Receive only their principal (after lockup period)
- Do not participate in winner selection
```

### 4. Late Deposits

Participants who deposit close to the draw date:
```
- Included in the current round
- Receive proportional winning chances
- Generate less yield due to shorter duration
```

**Example:**
- Alice deposits 1,000 USDC on day 0
- Bob deposits 1,000 USDC on day 6
- Draw on day 7

```
Y_Alice = 1,000 × 0.052 × (7/365) = 0.997 USDC
Y_Bob = 1,000 × 0.052 × (1/365) = 0.142 USDC
Y_total = 1.139 USDC

Prize pool = 1.139 USDC
P(Alice) = 50%
P(Bob) = 50%
```

Even though Bob deposited later and generated less yield, both have equal winning chances based on deposit amounts, not contribution to yield.

### 5. Rounding Errors

With many small deposits, rounding can accumulate:
```
Example: 1,000 deposits of 1 USDC each
Total: 1,000 USDC
Yield: 0.0001 USDC per deposit
Calculated: 0.1 USDC
Actual: 0.0999 USDC (due to truncation)
Error: 0.0001 USDC (0.1%)
```

**Mitigation:** Accumulate dust in protocol reserve for future rounds.

## Summary

The VaultQuest yield distribution mechanism is mathematically sound and economically fair:

1. **Proportional Chances**: Winning probability is directly proportional to deposit size
2. **Expected Value Preservation**: Average returns match the underlying protocol APY
3. **Principal Protection**: All deposits are guaranteed to be withdrawable after lockup
4. **Zero-Sum System**: Total yield generated equals total prizes distributed
5. **Transparent Calculation**: All formulas are deterministic and verifiable on-chain

The system creates an engaging savings experience through gamification while maintaining the mathematical properties of traditional yield-bearing accounts on average.
